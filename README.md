# Sports Analytics Innovation Project

## Project Overview
This project proposes a Player Workload and Availability Dashboard designed to support decision-making within a professional basketball organization. The idea is to create a simple, visual tool that summarizes player fatigue, injury risk, and recent workload trends in a way that is easy for coaches and front office staff to interpret. Rather than introducing a completely new metric, this project focuses on improving how existing performance and health data are organized and communicated to decision makers.

## Decision-Making Problem
Teams often struggle to balance player performance with injury prevention. Coaches must decide which players should rest or have limited minutes, how to adjust practice intensity, and whether a player is ready to return from injury. Currently, these decisions are often based on scattered reports or subjective judgment. This creates inconsistency and increases the risk of poor decisions.

## Proposed Analytics Approach
The dashboard would combine multiple data sources, including player minutes played, practice intensity data, injury history, recovery status, and physical tracking data such as movement and fatigue indicators. The analysis would focus on identifying patterns and summarizing them into simple categories such as Low Risk, Moderate Risk, and High Risk. The goal is not to build a complex predictive model, but to translate data into actionable insights.

## Use by Decision Makers
Coaches and performance staff would use this dashboard before games and practices. For example, a coach could quickly see which players are at high risk and reduce their minutes. A performance manager could recommend rest or modified training. Front office staff could also monitor long-term player health trends. The dashboard would be designed to support fast, practical decisions rather than technical analysis.

## Connection to Chapter 7
This idea currently represents the Creative Phase of innovation. At this stage, the focus is on clearly defining the problem, proposing a possible solution, and identifying how the idea could support decision-making. No testing or refinement has been done yet, which means the idea still needs to be prototyped and evaluated before adoption.
